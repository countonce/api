    Introduction

    Welcome to the CountOnce API documentation.

    Authentication

    To authenticate, use this code:

    import CountOnce from 'countonce-node';

const countOnce = new CountOnce({
    account_id: '<account_id>',
    auth_token: '<your api auth token>'
});

    <?php
use CountOnce\CountOnce;

$countOnce = new CountOnce([
    'account_id' => '<account_id>',
    'auth_token' => '<your api auth token>'
]);

    # With shell, you can just pass the correct header with each request
curl 'http://<account_id>.countapi.com/ping' \
  -H 'Content-Type: application/x-www-form-urlencoded; charset=UTF-8' \
  -H 'Authorization: Bearer <auth_token>'

    Make sure to replace <auth_token> with your API auth token.

    CountOnce expects the API key to be included in all requests to the server in a header that looks like the following:

    Authorization: Bearer <auth_token>

    You must replace auth_token with your API auth token.

    Installation

    Client libraries

    Install the client library for your chosen language, or interact directly via HTTPS request.

    npm install countonce-node

    composer require countonce/countonce-php

    Ping

    Send a ping

    import CountOnce from 'countonce-node';

const countOnce = new CountOnce({
    account_id: '<account_id>',
    auth_token: '<your api auth token>'
});

countOnce.ping({
    key: '<key>',
    attributes: {
        account_id: '<attribute value>',
        action: '<attribute value>',
    },
    unique_value: '<unique id or hash>',
    revenue: 0.00
}).catch(err => console.error(err));

    <?php
use CountOnce\CountOnce;
use Psr\Http\Message\ResponseInterface;
use GuzzleHttp\Exception\RequestException;

$countOnce = new CountOnce([
    'account_id' => 'account1',
    'auth_token' => '<your api auth token>'
]);

$promise = $countOnce->ping([
    'key' => '<key>',
    'attributes' => [
        'account_id' => '<attribute value>',
        'action' => '<attribute value>',
    ],
    'unique_value' => '<unique id or hash>',
    'revenue' => 0.00
]);

$promise->then(
    function(ResponseInterface $res) {
        print_r($res->getBody()->getContents());
    },
    function(RequestException $e) {
        echo $e->getMessage();
    }
);

// Wait for response before continuing
$promise->wait();

    curl 'http://account1.countapi.localhost:8000/ping' \
  -H 'Content-Type: application/x-www-form-urlencoded; charset=UTF-8' \
  -H 'Authorization: Bearer <auth_token>' \
  -d key=<key> \
  -d attributes[account_id]=<attribute value> \
  -d attributes[action]=<attribute value> \
  -d unique_value=<unique id or hash>
  -d revenue=0.00

    The above command returns a JSON response as follows. Client libraries always return Promises so you can send multiple requests asynchronously.

    ['OK']

    Send a ping request each time you want to count an event.

    Query Parameters

    Parameter Mandatory Description
    key Yes The name of the key
    unique_value Yes The unique value, for example a customer ID or IP address
    attributes Yes, for composite keys A dictionaty of key/value pairs, one for each attribute
    revenue No Revenue or numeric value associated with the event
    timezone No Use a different timezone to the one configured in the account
    Pings may fail with an error if constraints are in place.

    Queries

    Uniques

    import CountOnce from 'countonce-node';

const countOnce = new CountOnce({
    account_id: '<account_id>',
    auth_token: '<your api auth token>'
});

const queryOptions = {
    metric: '<metric>',
    filter: {
        'account_id': '1001'
    }
};

countOnce.getUniques('<key>', queryOptions).then(data => {
    for(let item in data.items) {
        console.log(data.items[item].attributes);
        for(let metric in data.items[item].uniques.range) {
            console.log(`- ${metric}: ${data.items[item].uniques.range[metric]}`);
        }
    }
}).catch(err => console.error(err));

    <?php
use CountOnce\CountOnce;
use Psr\Http\Message\ResponseInterface;
use GuzzleHttp\Exception\RequestException;

$countOnce = new CountOnce([
    'account_id' => 'account1',
    'auth_token' => '<your api auth token>'
]);

$promise = $countOnce->getUniques('<key>', [
    'metric' => '<metric>',
    'filter' => [
        'account_id' => 1001,
    ],
]);

$promise->then(
    function(ResponseInterface $res) {
        $result = json_decode($res->getBody()->getContents());
        foreach($result->items as $item) {
            print_r($item->attributes);
            foreach($item->uniques->range as $metric => $count) {
                echo sprintf('- %s: %s', $metric, $count) . PHP_EOL;
            }
        }
    },
    function(RequestException $e) {
        echo $e->getMessage();
    }
);

// Wait for response before continuing
$promise->wait();

    curl 'http://<account_id>.countapi.localhost:8000/uniques/<key>/<metric>' \
  -H 'Authorization: Bearer <auth_token>' \
  -G

}

    For composite keys, the JSON response looks as follows:

    {
  "items": [
    {
      "attributes": {
        "account_id": "1001",
        "action": "login"
      },
      "uniques": {
        "range": {
          "2020-06-01": 0,
          "2020-06-02": 0,
          "2020-06-03": 0,
          "2020-06-04": 0,

          "2020-06-06": 0,

        },
        "current": 0,
        "previous": 0,
        "total": 4
      }
    }
  ],
  "next_iterator": 0
}

    The uniques endpoint returns the number of unique ping values seen over time.

    URL Parameters

    Parameter Mandatory Description
    key Yes The name of the key
    metric No The metric to slice the date range. Valid options: daily, weekly, monthly, quarterly and annually.
    range No One of last7days, last14days, last30days, today, thismonth or lastmonth.
    start_date No The start date, if no range is set. Format YYYY-MM-DD
    end_date No The end date, if no range is set. Format YYYY-MM-DD
    prev_range No One of prev7days, prev14days, prev30days, yesterday, prevmonthtodate or prevmonth.
    prev_start_date No The previous start date, if no range is set. Format YYYY-MM-DD
    prev_end_date No The previous end date, if no range is set. Format YYYY-MM-DD
    include No Optional features to include in a comma-delimited list. Values are current (total for current period), previous (total for previous period), total (total for all time) and prev_range (sliced data by metric for the previous period).
    Note that the values under range will vary depending on the metric requested.

    Errors

    CountOnce uses the following error codes:

    Code Meaning
    401 Unauthorized -- Your API auth token is wrong.
    422 Invalid request -- parameters are missing or incorrect.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
